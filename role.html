第5章: Roleの作成

5.1: Roleのディレクトリ構造

Roleを使ってタスクを整理し、再利用可能な構造にします。まず、`httpd`を管理するRoleを作成します。

LinuxコマンドでRoleのディレクトリを作成

手動でRoleのディレクトリ構造を作成する場合、以下のコマンドを実行します。


$ mkdir -p roles/httpd/{tasks,handlers,templates,files,vars,defaults,meta}
$ touch roles/httpd/tasks/main.yml
$ touch roles/httpd/handlers/main.yml
$ touch roles/httpd/vars/main.yml
$ touch roles/httpd/defaults/main.yml
$ touch roles/httpd/meta/main.yml

これで、`httpd` Roleの基本的な構造が完成します。

ディレクトリ構造


roles/
└── httpd/
    ├── tasks/
    │   └── main.yml
    ├── handlers/
    │   └── main.yml
    ├── templates/
    ├── files/
    ├── vars/
    │   └── main.yml
    ├── defaults/
    │   └── main.yml
    ├── meta/
    │   └── main.yml

roles/httpd/tasks/main.yml の内容

作成した `main.yml` に、Apache（httpd）のインストールと起動のタスクを記述します。


- name: Install Apache
  ansible.builtin.package:
    name: httpd
    state: present

- name: Start and enable Apache
  ansible.builtin.service:
    name: httpd
    state: started
    enabled: yes

これにより、Apacheがインストールされ、サービスが起動して有効化されます。
